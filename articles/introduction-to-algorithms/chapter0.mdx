在开始编写这个博客时，我自学编程已经有近年半的时间了，当然其中有很多懒散和琐事缠身的时光，虽然也实习了快半年时间，但是对计算机基础的了解其实还很一般。很多人建议我看视频、报培训班来学习，可是我却是喜欢看书。我尝试啃一些大部头的书，例如 CSAPP、《现代操作系统》、《计算机网络》等等，这些书都非常厚，相信对于初学者足够枯燥。我并不是一个自制力强的人，常常犯懒，这些书要么是看着看着就吃灰了，要么就是匆匆翻阅了事。

在今天，我作为一名前端程序员已经工作满三个月了。三个月的实际工作经历，我感觉可以说自己已经快摸到了初级程序员的门槛，不可再以自己是一个初学者为接口，逃避这些大部头的书籍了。在程序员领域，前端工程师们们有时被算法工程师和后端程序员们轻视，认为他们只会堆砌系统，写写业务代码，实现些简单功能，而计算机基本功不扎实，缺乏深入思考问题和解决难题的能力。很遗憾地说，确实有不少前端程序员是这样的，甚至在其他程序员类别中也很常见，当然我现在工作的公司也有不少优秀的前端工程师。我希望借助这个博客的机会，能够约束自己，治一治自己的懒病，提升自己的计算机基本功，做个合格的程序员。也就是三年前，我像没头苍蝇一样选择考研，面对现在看来相当枯燥的经济学书籍也愿意花时间品读。因此我相信自己还有坚持阅读的能力，不至于在三年里面就完全退化了。

# 阅读《算法导论》

曾经读过读一本书，叫做[《How to Read a Book》](https://book.douban.com/subject/1013208/)，当时只是为了增强英语所以看了英文版。那边书其实内容不算多，但是讲得很细，其中作者反复强调读书的层次性。

> 每本书的封面之下都有一套自己的骨架，作为一个分析阅读的读者，责任是要找出这个骨架。
> 一本书出现在面前时，肌肉包着骨头，衣服包裹着肌肉，可说是盛装而来。
> 读者用不着揭开它的外衣或是撕去它的肌肉来得到在柔软表皮下的那套骨架，
> 但是一定要用一双 X 光般的透视眼来看这本书，因为那是了解一本书、掌握其骨架的基础。

![](https://tva1.sinaimg.cn/large/008i3skNgy1gy8wk9154gj30ci0jdjs4.jpg)

如果我开足耐心，那么我会选择从《算法导论》开始阅读，因为它足够重要、也足够厚重。面对这样的书籍，我从一开始就应该进入阅读的第二个层次“检视阅读”。因为毫无疑问这一本需要耐心和抗压能力去阅读的大部头经典，所以我应该很快过渡到“分析阅读”的层次，最后，这本章节繁多的著作，每一章的主题都值得我去进行“主题阅读”。

# 本书的作者和反响

如果你读过经济学的著作，那么一定会养成读书前必须仔细了解作者的习惯。因为亚当·斯密和大卫·李嘉图的对经济学的阐述肯定不能完全一样、哈耶克和凯恩斯在宏观经济方面的观点必然也截然相反、一本政治经济学的书和一本数理经济学的书读起来也是天差地别。读任何一本严肃的著作，一定要认识作者是谁，他的经历如何，他有哪些观点，他在该领域的成绩怎样？最后，这本书的评价怎样？

在阅读《现代操作系统》时，我曾经深深佩服这位伟大的作者，能够把一本讲解枯燥操作系统原理的书写得足够有趣，让最没有耐心的人也能够一看几个小时。后来我去查看了作者 Tanaenbaum，发现了他原来是 MINIX 系统的作者，著名的操作系统专家，而且和 GNU Linux 的创造者林纳斯对微内核和宏内核的分歧还有一场有趣的世纪论战。

本书《算法导论》的作者有四位，Cormen、Leiserson、Rivest 和 Stein，他们每一个都是标准的学院派，资深的数学家或计算机科学家，都在学术领域、教育领域有着卓越的成就，并且每一位都终身致力于算法领域的研究。这本书作为教科书风靡全世界，许多着名大学的计算机系都用作算法课程的参考书。这本书也被广泛简称为 CLRS，使用了四位作者的姓的首字母，可见已经成为不朽经典。

![](https://tva1.sinaimg.cn/large/008i3skNgy1gy8ywc8ay4j30wc08qdgx.jpg)

但是同时读者可能就不要指望阅读这本书能够速成、能够多么生动有趣了。

我首先去阅读了一些评论，有人指出需要这是一本典型的美式教材，非常冗长，教授们喜欢长篇大论，把概念讲解地非常详细，对于详细到啰嗦这点我并不反感。有一例[书评](https://book.douban.com/review/1319527/)我觉得很有价值：

> 书中算法的伪代码描述并不重要，学习算法的根本是要找出问题本身的数学结构、
> 理解解决这种问题结构的算法和它的设计思想、掌握具有各种结构特征的算法工具才是学习算法的根本。
> 《算法导论》的最大成就在于它的选材，它筛选出来的结果，每一个都是计算机科学当之无愧的里程碑。

我看电影喜欢先看影评，虽然总避免不了带了些先入为主的观念，但是我总是忍不住要看。其实从文字表达的方式、结构很容易看出来某个人的观点是否与自己类似。我相信总体观点、文字表达习惯类似的人，对某一具体事物的认知大概率也相近。我决定就带着些先入为主的观念来阅读这本书，我觉得带着有经验读者的观点去阅读学习，收获的期望更大些。就像统计学上常用一种方法，先给出一个具有相合性（俗话可说大差不差的）“先验”的估计，然后基于后面获得的样本来纠正估计偏差，往往能用更少的数据和计算量得到比较准确的估计。

# 本书简介与前言

简介中注明了本书适合作为本科生学习数据结构、研究生学习算法的教材，这个信息实际上很重要，它告诉我们这本书的基础地方在于数据结构，对算法的理解和掌握则是本书阅读后更高层次的收获。本书分为八个部分：

- 基础知识
- 排序和顺序统计量
- 数据结构
- 高级设计和分析技术
- 高级数据结构
- 图算法
- 算法问题选编
- 数学基础知识

重点在于算法的分析和设计上。

本书是第三版，它的前言我非常喜欢：

> 在计算机出现之前，就有了算法。现在有了计算机，就需要更多的算法，算法是计算的核心。

这一句话很质朴，然而给人的思考可以很多。人类在有计算机之前，面对的计算问题很少、很简单；面对的数据也少、结构也很简单；传说远古时期结绳记事、使用贝壳作为货币，在很晚才发明了数字和基本运算。人类认识到简单的加减乘除计算是很晚的事情，认识到现在许多高中生都会的微积分都已是近代的事情了，而离散数学、数值分析则已经是现代科学范畴了。后来有了计算机的出现，人们的计算能力巨幅提高，可以求解规模难以想象的难题。计算机可以在毫秒级别内给出极其复杂数学问题的答案。这是人们从未想象过的事情，于是人们也野心也越来越大、想要求解的问题也越来越多，终于诞生了利用现代计算机去求解问题的系统方法。“算法”这门课程，为什么是计算机科学和软件工程的基本学问？我想这个答案是很容易由这句话得出的，因为现代计算机科学家、硬件/软件工程师们，本质上每天使用计算去解决只有用计算机才能解决的问题，因此学习如何使用计算机去算问题的解非常重要，如同婴儿学习走路、吃饭。任何一个程序员都应该掌握这门用计算机去求解问题的基本学问。

序言写道：对于本科生课程，可以选用每一章前几节的内容；研究生课程则可以完整讲授一章。另外每一章都是相对独立的。也就是说这本书非常适合用于主题阅读、可以单独阅读某一个章节，并且深入下去。

# 阅读哪些部分

综合书评和前言以及对作者的了解，我们不难发现：本书是一本适合参考和主题阅读的书（这种大部头著作通常不适合线性阅读）。因此可以在想看哪一章节的时候看哪一个章节（基础是前几小节），并且只要看完时候有所收获，与之前所阅读的知识能够结合，能够形成网络即可。对于数据结构的部分需要重点掌握，这是转变思维、形成计算机思维习惯的重要步骤。没有学习过计算机课程的人，习惯于高中、大学所学习的数学知识，习惯于处理实数和连续函数。而这种思维实际上是不能套用到计算机处理问题的方法中的，因为计算机处理的数据几乎全部是离散的，每一个算法都可以拆分成可列个具体的、不可分割的步骤完成。这种思维转变方式有些跳跃，因此需要深刻认识到算法这门学问要解决哪些问题？能够运算哪些数据结构？算法研究的核心，是如何花费最小代价解决给定规模、给定条件的计算问题，并且计算问题必须要具体到可以进行运算的具体对象，这个对象必须是计算机可以处理的数据结构，而每一种算法都会针对某一种的数据结构。有时候明确数据结构是最重要的一个步骤、任何一个好的算法，一定是选取了最合适这个问题的数据结构，并且采用了最优的策略去操作它，从而在某一个可以测度的标准下实现了成本最小化。
